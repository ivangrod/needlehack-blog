Satel Integra

The satel_integra integrationIntegrations connect and integrate Home Assistant with your devices, services, and more. [Learn more] will allow Home Assistant users who own a Satel Integra alarm panel to leverage their alarm system and its sensors to provide Home Assistant with information about their homes. Connectivity between Home Assistant and the alarm is accomplished through a ETHM extension module that must be installed in the alarm. Compatible with ETHM-1 Plus module with firmware version > 2.00 (version 2.04 confirmed).

There is currently support for the following device types within Home Assistant:

  • Binary sensor: Reports on zone or output statuses
  • Switch: allows for setting states of selected outputs
  • Alarm control panel: represents the zones (in Polish: ‚Äústrefa‚Äù). Reports its status, and can be used to arm/disarm the partition

The module communicates via Satel‚Äôs open TCP protocol published on their website. It subscribes for new events coming from alarm system and reacts to them immediately.

Setup

Please note that ETHM-1 module is currently not supported: it does not provide functionality used by this extension. At the moment only ETHM-1 Plus module is supported. That might change in the future, but no promises are given.

The library currently doesn‚Äôt support encrypted connection to your alarm, so you need to turn off encryption for integration protocol. In Polish: ‚Äúkoduj integracje‚Äù must be unchecked. You will find this setting in your DloadX program.

A list of all partition, zone and output IDs can be acquired by running DloadX program and connecting to your alarm.

For the binary sensor, check the type/class list for a possible visualization of your zones. Note: If no zones or outputs are specified, Home Assistant will not load any binary_sensor integrations.‚Äù

  host: IP_ADDRESS

Configuration Variables

Looking for your configuration file?
host string Required, default: localhost

The IP address of the Satel Integra ETHM module on your home network, if using socket type.

port integer (Optional, default: 7094)

The port on which the ETHM module listens for clients using integration protocol.

code string (Optional)

User password, it‚Äôs needed for making use of the switchable_outputs. It‚Äôs recommended not to use admin password.

partitions integer | list (Optional)

List of the partitions to operate on.

name string Required

Name of the partition.

arm_home_mode integer (Optional, default: 1)

The mode in which the partition is armed when ‚Äòarm home‚Äô is used. Possible options are 1,2 or 3. For more information on what the differences are between them, please refer to Satel Integra manual.

zones integer | list (Optional)

This parameter lists the zones (or inputs) that will be visible by Home Assistant. For each zone, a proper ID must be given as well as its name. The name is arbitrary and does not need to match the one specified in Satel Integra alarm configuration.

name string Required

Name of the zone.

type string (Optional, default: motion)

The zone type.

outputs integer | list (Optional)

Very similar to zones, but with outputs. Satel Integra uses outputs to inform external systems about different events. For example power failure, or that alarm started counting for exit or some other user-defined condition. They may be used for simple alarm-based automation. For more information please refer to Satel homepage and forums.

name string Required

Name of the output.

type string (Optional, default: motion)

The type of the device - just for presentation.

switchable_outputs integer | list (Optional)

Switchable outputs. These will show up as switches within Home Assistant.

name string Required

Name of the output.

Full examples 

# Example configuration.yaml entry
satel_integra:
  host: 192.168.1.100
  port: 7094
  partitions:
    01:
      name: "House"
      arm_home_mode: 2
    02:
      name: "Garage"
  zones:
    01:
      name: "Bedroom"
      type: "motion"
    02:
      name: "Hall"
      type: "motion"
    30:
      name: "Kitchen - smoke"
      type: "smoke"
    113:
      name: "Entry door"
      type: "opening"
  outputs:
    05:
      name: "Garden lights trigger"
      type: "light"
    09:
      name: "Gate opening trigger"
      type: "opening"
    30:
      name: "Alarm triggered"
      type: "safety"
    32:
      name: "Alarm power problem"
      type: "safety"
  switchable_outputs:
    05:
      name: "Gate open"
    06:
      name: "Gate close"    
    14:
      name: "Garden light"

Having configured the zones and the outputs, you can use them for automation, such as to react on the movement in your bedroom. For example:

  alias: "Flick the input switch when movement in bedroom detected"
  triggers:
    - trigger: state
      entity_id: "binary_sensor.bedroom"
      to: "on"
  actions:
    - action: input_boolean.turn_on
      target:
        entity_id: input_boolean.movement_detected
